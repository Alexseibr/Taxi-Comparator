# Деплой на VPS под доменом pzk.by

Сайт — это статический фронтенд (React, собранный Vite в HTML+JS+CSS). Никакой
БД, никакого Node-сервера на проде не нужно. Всё, что требуется — раздать
содержимое архива `pzk-by.tar.gz` через nginx.

## Что в этой папке

| Файл                  | Назначение                                                 |
| --------------------- | ---------------------------------------------------------- |
| `pzk-by.tar.gz`       | Готовая статика сайта (`index.html`, `assets/...`)         |
| `nginx-pzk.by.conf`   | Готовый nginx-конфиг для домена pzk.by                     |
| `deploy.sh`           | Скрипт первой выкатки (ставит nginx, certbot, заливает)    |
| `update.sh`           | Скрипт обновления (раскатить новый билд без даунтайма)     |

## Шаг 0. DNS

В панели вашего регистратора создайте две A-записи:

```
pzk.by      A    <IP вашего VPS>     TTL 300
www.pzk.by  A    <IP вашего VPS>     TTL 300
```

Подождите 5–15 минут пока DNS обновится. Проверить можно командой:

```bash
dig pzk.by +short
```

## Шаг 1. Залить файлы на VPS

С вашего компьютера:

```bash
scp -r deploy/pzk.by root@<IP-VPS>:/root/
```

(или используйте FileZilla / WinSCP — без разницы)

## Шаг 2. Запустить установку

Зайти на VPS по SSH:

```bash
ssh root@<IP-VPS>
cd /root/pzk.by
chmod +x deploy.sh update.sh
./deploy.sh pzk-by.tar.gz
```

Скрипт сделает следующее:

1. Поставит `nginx` и `certbot` (если их нет).
2. Создаст папку `/var/www/pzk.by` и распакует туда сайт.
3. Подключит конфиг `/etc/nginx/sites-enabled/pzk.by`.
4. Спросит подтверждение (DNS уже работает?) и выпустит SSL-сертификат
   Let's Encrypt на оба домена.
5. Включит автоматическое продление сертификата через `certbot.timer`.

После завершения сайт открывается на `https://pzk.by`.

## Шаг 3. Обновление сайта

Когда я пришлю новый `pzk-by.tar.gz`:

```bash
scp pzk-by.tar.gz root@<IP-VPS>:/root/pzk.by/
ssh root@<IP-VPS>
cd /root/pzk.by
./update.sh pzk-by.tar.gz
```

Скрипт:

1. Сохранит текущую версию рядом как `pzk.by-backup-YYYYMMDD-HHMMSS.tar.gz`.
2. Зальёт новый билд в `/var/www/pzk.by`.
3. Перезагрузит nginx без даунтайма.

## Шаг 4. Подкладывание новых замеров без редеплоя

В `/var/www/pzk.by/data/observations.json` лежит файл с дополнительными
точечными замерами сёрджа (например, со скриншотов Яндекса). Этот файл можно
**обновлять отдельно**, без новой сборки сайта — просто залить новый вариант:

```bash
scp observations.json root@<IP-VPS>:/var/www/pzk.by/data/
```

Никакого перезапуска nginx не требуется. Пользователю достаточно обновить
вкладку (Ctrl+F5).

Формат файла, схема и пример описаны прямо в архиве — `/var/www/pzk.by/data/README.md`.

> Внимание: при следующем `update.sh` файл `observations.json` будет заменён
> на тот, что я положил в архив. Если планируете править его на сервере,
> не забывайте сохранять копию или присылать мне ваш текущий файл, чтобы я
> включал его в новые билды.

## Шаг 5. Свои поездки в браузере

Пользователь может прямо на сайте нажать «Мои поездки» и подгрузить CSV или
JSON со своими данными — они сохранятся **в его браузере** (`localStorage`),
не уходят на сервер и тоже подмешаются в карту. Очистка — кнопкой в том же
диалоге.

Если что-то пошло не так — откатить можно одной командой:

```bash
cd /var/www
rm -rf pzk.by/* && tar -xzf pzk.by-backup-YYYYMMDD-HHMMSS.tar.gz -C pzk.by/
```

## Технические детали

- **Размер сайта:** ~750 КБ JS + ~110 КБ CSS (gzip-сжатие в nginx даёт ~250 КБ
  по сети).
- **Зависимости в проде:** только OSM-тайлы (`tile.openstreetmap.org`) и
  Nominatim (опциональный геокодер). API на VPS не нужен.
- **Минимальные требования к VPS:** 1 CPU / 512 МБ RAM / 5 ГБ диска. Подойдёт
  любой self-hosted Ubuntu/Debian/CentOS.
- **Кэш-стратегия:** файлы в `/assets/` имеют hash в имени и кэшируются на
  год; `index.html` всегда свежий (`Cache-Control: no-cache`), что гарантирует
  что новый деплой увидят все пользователи без F5.

## Если что-то не работает

```bash
# Логи nginx
tail -f /var/log/nginx/error.log

# Проверить что nginx видит сайт
curl -I http://pzk.by

# Проверить статус сертификата
certbot certificates

# Перезапустить nginx
systemctl reload nginx
```
