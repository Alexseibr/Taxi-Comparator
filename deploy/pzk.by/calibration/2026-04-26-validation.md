## Цель
Проверить нашу новую гипотезу `cmf_дисплей ≈ surge × минималку` на **новых** точках,
которые НЕ участвовали в подгонке.

## Решение по формуле

Гипотеза `Yandex_дисплей ≈ surge × min` валидирована на out-of-sample наборе (4.0%).

Возможные пути:
1. **Минимально-инвазивный фикс** для текущей модели: уменьшить `perKm = 0.05`,
   `perMin = 0.05` — тогда `raw < min` для всех городских поездок (<25 км),
   и формула де-факто работает как `surge × min`.
   - Плюс: меньше изменений, текущие тесты не сломаются.
   - Минус: «честный» расчёт перекилометража исчезнет → плохо для собственного RWB Taxi.

2. **Двойной режим**:
   - «Прогноз Yandex Go» = `surge × min` (для сравнения)
   - «RWB честный тариф» = старая формула (для запуска своего сервиса)

3. **Оставить как есть** + добавить визуальное предупреждение в UI: «При высоких
   сёрджах фактическая цена Yandex может быть ниже на 30-100%».
