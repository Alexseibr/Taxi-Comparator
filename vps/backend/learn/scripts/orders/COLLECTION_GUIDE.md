# Как собирать замеры (для сотрудников и админа)

Цель: за один сеанс снять **батч** скриншотов Yandex Go с одной точки А
на 5–10 разных направлений в фиксированный час. Из такого батча получается
сразу 5–10 калибровочных точек, и обучающая модель видит **зависимость
сёрджа от часа** — теперь она работает по часовому слоту, а не по 4-часовому.

---

## Шаг 1. Какие батчи собирать (план диспетчерской)

Делим Минск на 6 базовых районов сбора:

| Район-точка А                        | Когда снимать                       |
|--------------------------------------|--------------------------------------|
| Центр (пл. Победы / Кальварийская)   | 7:00, 8:00, 9:00, 13:00, 17:00, 18:00, 22:00 |
| Север (Веснянка / Дрозды)            | 7:00, 8:00, 13:00, 18:00            |
| Юг (Куприянова / Уборевича)          | 8:00, 9:00, 13:00, 17:00, 18:00     |
| Восток (Уручье / Партизанский)       | 8:00, 9:00, 17:00, 18:00            |
| Запад (Кунцевщина / Сухарево)        | 8:00, 9:00, 17:00, 18:00            |
| Северо-запад (Каменная Горка)        | 8:00, 9:00, 17:00, 18:00            |

**Минимальный целевой объём**: 1 батч на район в день в каждый из часов
из списка. Лучше — каждый час с 7:00 до 22:00 (тогда модель сама поймёт
утренний пик и обеденный спад).

---

## Шаг 2. Что именно делать сотруднику

1. Открыть Yandex Go из точки А.
2. **Один за другим** забить 5–10 направлений (см. шаблон района ниже).
3. Для каждого направления записать:
   - **цена Эконом** (br) — цифра под классом «Эконом»
   - **цена Комфорт** (br) — то же самое для класса «Комфорт»
   - **подача** (мин) — «через N мин» сверху экрана
   - **время в пути** (мин) — если показано на превью
   - **метка спроса** — 🟢 (без молнии) / 🟡 (молния, surge < 1.5) / 🔴 (молния, surge ≥ 1.5)
4. Если какого-то класса в момент замера нет (например, Эконома) — оставить пустым.

Все замеры за час уходят в один CSV или прямо в форму на сайте
(см. README артефакта).

---

## Шаг 3. Шаблоны маршрутов (`scripts/orders/templates/`)

Чтобы не вписывать адреса каждый раз, держим **шаблоны направлений** —
JSON-файлы с одной точкой А и списком стандартных точек Б.

Пример: `templates/west-from-kalvariyskaya.json`
```json
{
  "id": "west-from-kalvariyskaya",
  "name": "Из Кальварийской 17 на 5 западных направлений",
  "from": "Кальварийская 17",
  "fromCoords": [53.91, 27.52],
  "destinations": [
    { "to": "м. Пушкинская",     "coords": [53.892, 27.482] },
    { "to": "Куприянова 4",      "coords": [53.85,  27.50 ] },
    { "to": "ТЦ Galleria Minsk", "coords": [53.901, 27.564] },
    { "to": "пл. Якуба Коласа",  "coords": [53.917, 27.582] },
    { "to": "ж/д вокзал",        "coords": [53.890, 27.547] }
  ]
}
```

Заводим один шаблон на каждый район-точку А (пример — выше).
Главное — координаты должны быть точные (берём из Я.Карт), иначе OSRM/TomTom
посчитают чужой маршрут.

---

## Шаг 4. Импорт батча (для админа)

Сотрудник присылает CSV вида (имя колонок строго такое):

```csv
date,hour,destIdx,factE,factC,etaMin,demand
2026-04-27,14,0,12.5,15.0,5,green
2026-04-27,14,1,18.0,21.5,7,yellow
2026-04-27,14,2,9.5,11.0,4,green
2026-04-27,14,3,16.0,19.5,6,yellow
2026-04-27,14,4,21.0,25.0,9,red
```

`destIdx` — индекс направления в `destinations` шаблона (0-based).
`demand` — `green` / `yellow` / `red`.

Команда:

```bash
pnpm import-batch scripts/orders/templates/west-from-kalvariyskaya.json batch.csv
```

На выходе — `scripts/orders/manual-2026-04-27.json` (или `manual-2026-04-27-h14.json`,
если в одном файле несколько часов — сгруппируется автоматически), готовый
для штатного:

```bash
pnpm calib scripts/orders/manual-2026-04-27.json
pnpm learn
pnpm apply
```

---

## Шаг 5. Альтернатива — форма на сайте

Если сотрудник не хочет работать с CSV — он может вносить замеры через
форму **«Внести замер из Яндекса»** в модалке «Мои поездки» прямо на сайте.
Каждый замер сразу отображается на карте (тепловая прослойка пользовательских
наблюдений). После накопления — экспорт CSV → админ запускает
`pnpm import-trips file.csv` (старый сценарий, по одной точке).

Шаблонный сбор быстрее: 5 направлений за 1 минуту вместо 5 минут на
ввод адресов.
