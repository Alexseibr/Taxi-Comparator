# Overview

Taxi Compare is a web application that helps users find the most affordable ride across various ride-hailing services such as Uber, Bolt, Yandex Go, and Lyft. It provides a side-by-side comparison of estimated fares, sorted from cheapest to most expensive, with the best option highlighted. For Minsk, it includes a "Tariff grid" displaying real-time Yandex Go prices for popular routes. The project aims to offer comprehensive, real-time ride-hailing price comparisons to empower users in making informed travel decisions. The project also incorporates a sophisticated anti-fraud system, `Newstat`, to analyze order behavior and detect fraudulent activities, ensuring fair practices within the ride-hailing ecosystem.

## System Architecture

The project is a monorepo built with Node.js 24 and TypeScript 5.9, using pnpm workspaces.

**Frontend:**
-   **Technology Stack:** React, Vite, Tailwind CSS, wouter, TanStack Query, framer-motion.
-   **UI/UX:** Focuses on a clean, intuitive interface for presenting complex pricing data, including interactive maps and tariff grids.
-   **Code Splitting:** Implemented lazy loading for routes and vendor chunks to optimize initial load times.

**Backend (API Server):**
-   **Technology Stack:** Express 5, PostgreSQL (with Drizzle ORM), Zod.
-   **API:** Defined using OpenAPI specification, with client and server code generated by Orval.
-   **Pricing Model:** Calculates fares based on road distance, duration, base fare, per-kilometer/minute charges, service fees, and surge multipliers, with city-specific tariffs. Includes a CatBoost-based MultiQuantile price prediction model for Yandex Go fares, providing P10-P90 ranges.
-   **Yandex Go Scraper:** Gathers real-time price data, with retry and fallback mechanisms.
-   **Surge & Fleet Models:** Deterministic Gaussian peaks for surge forecasting and a simulation for taxi fleet distribution.
-   **Data Calibration:** Automates recording of frontend quotes and processes user-uploaded Yandex Go screenshots (via Gemini Vision Pro) to refine pricing models.
-   **WB Anti-Fraud Module (Newstat):** A separate analytical module for fraud detection and decision workflow.
    -   **Features:** Analyzes client/driver/pair behavior, identifies fraud signals (e.g., auto-sharing, fast cancellations, subsidy abuse), enriches identity data, and provides a workflow for anti-fraud agents to review risks and make decisions.
    -   **ML Integration:** Incorporates a CatBoost-based ML service for weak-supervised and human-labeled supervised fraud scoring, with features like batch prediction and SHAP explanations.
    -   **Tariff Comparison:** Provides an endpoint for honest cross-source comparison of WB orders vs. Yandex Go calibration data.
    -   **Screenshot Management:** Handles mobile app screenshot uploads, including operator attribution, client-side image compression (WebP), and SHA-256 deduplication.

## External Dependencies

-   **OpenStreetMap Nominatim:** Address autocomplete and geocoding services.
-   **Google Maps Platform:** Routes API, Places Text Search, and Geocoding API for routing, autocomplete, and reverse geocoding.
-   **TomTom API:** Fallback for traffic data.
-   **HERE Technologies API:** Secondary fallback for traffic data.
-   **Yandex Go API:** Public endpoints for real-time tariff data.
-   **PostgreSQL:** Primary relational database.
-   **Gemini Vision Pro (Google AI):** Used for processing mobile app screenshots for calibration data and fraud detection.
