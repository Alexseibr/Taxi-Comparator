# Папка данных сайта

В этой папке лежат файлы, которые **подгружаются клиентом во время работы сайта**
и могут обновляться **без пересборки и редеплоя кода** — достаточно положить
новый файл в `/var/www/pzk.by/data/` и перезагрузить страницу.

## `observations.json`

Дополнительные точечные замеры сёрджа (со скриншотов Яндекс Go или из других
источников). Каждый элемент массива `items` — это одна точка с координатами,
днём недели, временным слотом и значениями сёрджа.

### Минимальный пример

```json
{
  "version": 1,
  "updatedAt": "2026-04-25",
  "items": [
    {
      "id": "yandex-2026-04-25-pobed",
      "lat": 53.9105,
      "lng": 27.5142,
      "day": "saturday",
      "slot": "evening",
      "comfortSurge": 2.2,
      "economSurge": 1.0,
      "hiddenEconomSurge": 2.6,
      "date": "2026-04-25",
      "source": "yandex-screenshot",
      "address": "пр. Победителей, 65",
      "notes": "Скриншот: Comfort ×2.2, Эконом ×1.0 но цена выше расчётной"
    }
  ]
}
```

### Поля

| Поле                | Обяз. | Что значит                                           |
| ------------------- | :---: | ---------------------------------------------------- |
| `id`                | да    | Уникальная строка, по ней можно потом найти/удалить. |
| `lat`, `lng`        | да    | Координаты точки.                                    |
| `day`               | да    | `weekday` / `saturday` / `sunday`                    |
| `slot`              | да    | `night` / `morning` / `midday` / `evening` / `late`  |
| `comfortSurge`      | нет   | Видимый множитель класса Comfort (например 1.7).     |
| `economSurge`       | нет   | Видимый множитель Эконом (обычно 1.0).               |
| `hiddenEconomSurge` | нет   | «Скрытый» эконом-сёрдж, посчитанный из цены.         |
| `date`              | да    | Когда замер сделан, `YYYY-MM-DD`.                    |
| `source`            | нет   | Произвольная метка источника.                        |
| `notes` / `address` | нет   | Свободный текст.                                     |

### Как это влияет на карту

Каждое наблюдение становится **дополнительной точкой** в IDW-интерполяции с
тем же весом, что и встроенные зоны, но с трёхкратным бонусом — то есть рядом
с замером прогноз «подтягивается» к реальному значению.

### Как обновить на VPS

```bash
scp observations.json root@<IP-VPS>:/var/www/pzk.by/data/
```

Никакого перезапуска nginx не нужно — файл сразу станет доступен.
Пользователю достаточно обновить вкладку (Ctrl+F5).

## Поездки пользователя

Свои поездки можно загружать **прямо в браузере** через кнопку «Мои поездки» в
шапке сайта — они сохраняются в `localStorage` и не уходят на сервер. Формат
там тот же самый (CSV или JSON), описание есть в самом диалоге.
