## Как замерять (15 минут на цикл)

1. **Открыть Yandex Go** на телефоне в нужный час.
2. **Выбрать 5–8 точек** из списка маршрутов выше. Старайтесь брать разные районы и длины.
3. **Для каждого маршрута**:
   - Сохранить скриншот с открытым ⚡N (`Cmf 1.5`, `Эконом 12.9`, мин ожидания).
   - Записать в JSON: `factC`, `factE`, `yaSurgeC`, `hour`, координаты «откуда».
4. **Сохранить** файл `scripts/orders/2026-MM-DD-HHMM.json` (см. README в той же папке).
5. **Запустить** `pnpm calib && pnpm learn`.

После 5–10 замеров в `weekday-morning` слой L4 в `traffic-effect.json` покажет реальную корреляцию `surge ↔ ttMult`, и `trafficSurgeMultiplier` можно будет уточнить (сейчас угол наклона задан вручную как 0.80/0.85).

## Что точно НЕ требуется замерять

- Время суток < 06:00 — Yandex чаще всего не активен, толку мало.
- Маршруты внутри одной зоны < 1 км — вырождается в minimum × сёрдж.
- Замеры с разрывом < 5 минут на одной точке — дублирование.
